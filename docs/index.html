AlgoLearn

An interactive platform for learning algorithms through step-by-step visualizations and hands-on coding challenges. Runs entirely on your machine.

TypeScript React 19 D3.js Monaco Editor Fastify Docker Sandbox
Overview

What does AlgoLearn do?

AlgoLearn combines three things that make algorithm learning click: watching them run, writing your own code, and getting instant feedback — all in one place.

Watch Algorithms Run

Step through sorting, graph traversal, dynamic programming, and pathfinding algorithms with animated visuals. Pause, rewind, speed up — you're in control.

Write & Test Code

Solve problems in a professional code editor with TypeScript support. Submit your solution and see test results instantly — just like a real coding interview.

Track Your Progress

See which problems you've solved, attempted, or haven't tried yet. The dashboard gives you a bird's-eye view of your learning journey.

Architecture

How the pieces fit together

Three services work in concert. The browser talks to the API, which talks to Docker when it needs to run your code safely.

Frontend

Built with React 19 and Vite. Algorithm visualizations are powered by D3.js, the code editor uses Monaco (the engine behind VS Code), and state is managed with Zustand. Styled with Tailwind CSS.

API

A Fastify server exposes REST endpoints for problems, submissions, and progress. Data lives in a local SQLite database managed through Drizzle ORM. No external database setup needed.

Runner

Your submitted code runs inside a Docker container that is locked down tight — no internet, limited memory, and auto-destroyed after each run. It's as safe as it gets.

How visualizations work under the hood

Generator Function Pure algorithm logic
yields →
Step Array [Step₁, Step₂, … Stepₙ]
drives →
Playback Hook Index + controls
renders →
D3.js Canvas Animated SVG

  1. Generator captures every step

    Each algorithm is written as a JavaScript generator function. At every meaningful moment — a comparison, a swap, a node visit — the function yields a complete snapshot of its current state (the array, which elements are being compared, which are sorted, etc.).

  2. Steps are collected into an array

    When you click "Run", the generator is consumed eagerly and all snapshots are stored in an array. Because the data is small (a few hundred steps for typical inputs), this is near-instant.

  3. Playback hook controls the timeline

    A React hook tracks the current step index, play/pause state, and speed. Playing simply increments the index on a timer. Going backward? Just decrement. Jump to any step? Set the index. No undo logic required.

  4. D3.js animates between states

    Each visualizer component receives the current step and uses D3's data-join pattern to smoothly transition SVG elements — bars slide into position, nodes change color, grid cells light up. React owns the <svg> container; D3 owns everything inside it.

Code Execution

How your code gets run

When you hit "Submit", your code takes a short but carefully guarded journey from the editor to the results panel.

  1. You write your solution

    The Monaco editor gives you a professional TypeScript editing experience — autocomplete, syntax highlighting, error squiggles — right in the browser.

  2. Your code is sent to the API

    The frontend sends your code and the problem ID to the API server over a simple HTTP request. The API looks up the problem's test cases from the database.

  3. A sandbox container is created

    The API writes your code to a temporary directory and spins up a brand-new Docker container with strict security settings (more on that below). Your code is mounted read-only.

  4. The test harness runs your code

    Inside the container, a test harness imports your function, feeds it each test case's input, compares the output to the expected answer, and measures execution time.

  5. Results come back

    The container outputs structured JSON with pass/fail for each test case. The API parses this, saves the submission to the database, updates your progress, and sends the results to the browser. The container is destroyed.

Security

The sandbox keeps things safe

Running user code is inherently risky. AlgoLearn uses multiple layers of Docker-based isolation so submitted code can't harm your machine.

No network Container cannot reach the internet or any other service
Read-only filesystem Only a tiny 16 MB temp area is writable
Resource limits 128 MB RAM, half a CPU, 64 processes max
10-second timeout Container is killed after 10 seconds no matter what
Non-root user Code runs as an unprivileged user with no-new-privileges
Auto-cleanup Container is force-removed and temp files deleted after every run
API

REST endpoints

The API is straightforward — seven endpoints cover everything the frontend needs.

Method Endpoint What it does
GET /api/categories Lists all algorithm categories (Sorting, Graph, DP, Pathfinding)
GET /api/problems Lists problems, optionally filtered by category or difficulty
GET /api/problems/:slug Fetches a single problem with its description, starter code, and visible test cases
POST /api/submissions Submits your code — runs it in the sandbox and returns pass/fail results
GET /api/submissions/:id Retrieves the result of a previous submission
GET /api/progress Returns your solve status for all problems
GET /api/health Health check — returns {"status":"ok"}
Project Structure

What's in each package

The project is a pnpm monorepo with four packages. Here's a map of what lives where.

shared Type Definitions

Problem, TestCase, Submission, and UserProgress types. Visualization step interfaces for all four algorithm categories. API request/response DTOs. Shared by both frontend and API.

frontend User Interface

algorithms/ — 10 generator functions
components/ — Dashboard, problem list, editor, 4 D3 visualizers, playback controls
hooks/ — useVisualization playback engine
stores/ — Zustand for problems & visualization state

api Backend Server

routes/ — Problems and submissions endpoints
services/ — Docker code runner orchestration
db/ — Drizzle schema, SQLite migrations
seed/ — 9 built-in problems with test cases

runner Execution Sandbox

A minimal Node.js Alpine Docker image. Contains a test harness that imports the user's function, runs it against test cases, catches errors, measures time, and outputs JSON. Stripped of everything unnecessary.

Getting Started

Run it on your machine

Everything runs locally. No accounts, no cloud services, no subscriptions. Just your machine, Node.js, and Docker.

Option A — Development mode

# Install dependencies
pnpm install

# Build the sandbox image (one time)
docker build -t algo-learn-runner:latest packages/runner

# Seed the database with problems
pnpm --filter api db:seed

# Start dev servers with hot reload
pnpm run dev

Frontend at localhost:3000  |  API at localhost:3001

Option B — Docker Compose

# Build and start everything
docker compose up --build

Frontend at localhost:3000  |  API at localhost:3001

Prerequisites

Node.js 20+, pnpm 9+, and Docker must be installed.

CI / CD

Automated quality checks

Every push and pull request to main triggers a GitHub Actions pipeline with three jobs.

Lint & Typecheck

ESLint and tsc --noEmit across all packages. Catches style issues and type errors before they merge.

Unit Tests

Vitest runs across all packages. Algorithm generators, API routes, and components are all covered.

Integration Tests

Builds the runner image and runs end-to-end tests that submit code through the API into real Docker containers.
